Local Qwen Home Computer - recent fixes

v2.9.1
- Windows setup bootstrap vise ne zavisi od `powershell.exe` iz `PATH`, nego koristi eksplicitnu sistemsku putanju do Windows PowerShell-a, pa instalacija vise ne puca sa `exit code 9009` na masinama sa netipicnim `PATH`-om
- Windows launcher wrapper-i i skriveni `.vbs` starteri sada koriste istu pouzdanu PowerShell putanju, tako da `Control Center`, repair i helper launcheri vise ne zavise od shell okruzenja korisnika
- Windows `repair-install`, `start-opencode`, `launch-agent`, `test-prompt` i `control-center` tokovi su poravnati na isti PowerShell lookup, da se isti problem ne vrati posle instalacije ili update-a

v2.9.0
- Dodat je pravi `Uninstall Local Qwen` tok na Windows strani, sa izborom da se obrisu samo precice, da se aplikacija ukloni ali modeli sacuvaju, ili da se obrise sve ukljucujuci modele (default)
- Windows installer sada pravi i `Uninstall Local Qwen` desktop precicu, pa uninstall ne zavisi od rucnog trazenja skripti ili foldera
- `ninja`, `cmake`, CUDA i VS Build Tools vise nisu hard blocker za osnovnu Windows instalaciju; ako nedostaju i ne mogu da se instaliraju, installer jasno prijavljuje da ce TurboQuant biti preskocen, ali lokalni `llama.cpp` fallback ostaje upotrebljiv

v2.8.5
- `WDAC / App Control` warning se sada automatski tretira kao istorijski ako je `llama-server` vec aktivan i health endpoint vraca `ok`, pa healthy masine vise ne dobijaju lazni aktivni alarm
- Windows i Linux `verify-install` sada prikazuju sanitizovan `install report`, tako da zastareli WDAC warning ne prekriva stvarno zdravo stanje sistema
- Health i repair tokovi sada koriste iste filtrirane warning podatke, pa `Repair App Control` vise ne iskace kao primarni problem kada server vec normalno radi

v2.8.4

- Dodat je `Install update` tok: Windows `Control Center` sada ima posebno dugme za pokretanje latest update installera, a Linux control center ima novu opciju `Install update`
