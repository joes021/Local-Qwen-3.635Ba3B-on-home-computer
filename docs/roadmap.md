# Roadmap

This roadmap follows one strict order. Each item below is intentionally rated at `10/10` because every one of them is important for turning the project from a working prototype into a stable product.

## Upgrade order

1. Installer self-healing
⭐⭐⭐⭐⭐⭐⭐⭐⭐⭐

2. Real post-install validation
⭐⭐⭐⭐⭐⭐⭐⭐⭐⭐

3. Central log viewer
⭐⭐⭐⭐⭐⭐⭐⭐⭐⭐

4. Stronger hardware profiling
⭐⭐⭐⭐⭐⭐⭐⭐⭐⭐

5. Full release automation
⭐⭐⭐⭐⭐⭐⭐⭐⭐⭐

## v1.3.0

Focus:

- installer self-healing
- real post-install validation

What this version is meant to improve:

- safer reruns on partially completed installs
- clearer detection of missing runtime, model, and config pieces
- explicit repair-oriented reporting instead of silent partial success
- stronger verification output on both Windows and Linux

## v1.4.0

Focus:

- central log viewer
- stronger hardware profiling

What this version is meant to improve:

- easier diagnosis from one place
- clearer hardware summary
- better automatic profile recommendations
- more transparent runtime plan selection
- cross-platform visibility into latest runtime and install logs

## v1.5.0

Focus:

- full release automation

What this version is meant to improve:

- fewer manual release steps
- stricter cross-platform release consistency
- automatic release integrity checks
- more predictable Windows and Linux publishing
- one-command local cross-platform release flow
